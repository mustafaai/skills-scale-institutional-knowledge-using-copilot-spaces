## Purpose
Provide role-specific onboarding checklists to ensure new team members can quickly understand their responsibilities, key tools, and collaboration patterns within OctoAcme projects.

---

## General Onboarding (All Roles)

- [ ] Complete company-wide onboarding and security training
- [ ] Access granted to required systems and tools
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles & Personas](octoacme-roles-and-personas.md)
- [ ] Introduced to team and key stakeholders
- [ ] Added to relevant communication channels (Slack, email lists)
- [ ] Assigned onboarding buddy or mentor
- [ ] Review current active projects and roadmap

---

## UX/UI Designer Onboarding

- [ ] Access to design tools (Figma, Sketch, Adobe Creative Suite)
- [ ] Review design system and component library
- [ ] Review accessibility standards (WCAG guidelines)
- [ ] Introduction to Product Manager and key stakeholders
- [ ] Shadow design review sessions
- [ ] Review user research repository and past findings
- [ ] Access to analytics tools for user behavior insights
- [ ] Review collaboration workflow with developers
- [ ] Understand design handoff process and documentation standards
- [ ] Review recent design decisions and rationale

---

## Data Analyst Onboarding

- [ ] Access to data warehouse and BI tools (Tableau, Looker, etc.)
- [ ] Review data governance and privacy policies
- [ ] Access to analytics platforms (Google Analytics, Mixpanel, etc.)
- [ ] Review existing dashboards and KPIs
- [ ] Introduction to Product and Engineering leads
- [ ] Review data schema and instrumentation documentation
- [ ] Access to SQL query tools and data notebooks
- [ ] Review reporting cadence and stakeholder expectations
- [ ] Understand data quality standards and validation processes
- [ ] Shadow weekly metrics review meetings

---

## Release Manager Onboarding

- [ ] Access to version control systems and release branches
- [ ] Review release cadence and scheduling process
- [ ] Access to CI/CD pipelines and deployment tools
- [ ] Review [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Introduction to Engineering, Product, and Support leads
- [ ] Review rollback procedures and incident response playbook
- [ ] Access to release communication channels
- [ ] Review release notes templates and examples
- [ ] Understand go/no-go decision criteria
- [ ] Shadow at least one complete release cycle
- [ ] Review post-release verification procedures

---

## Support Lead Onboarding

- [ ] Access to support ticketing system (Zendesk, Jira Service Desk, etc.)
- [ ] Review support SLAs and escalation procedures
- [ ] Access to customer communication channels
- [ ] Review knowledge base and support documentation
- [ ] Introduction to Product, Engineering, and Release Manager
- [ ] Review common customer issues and resolutions
- [ ] Access to internal troubleshooting tools
- [ ] Shadow experienced support team members
- [ ] Review incident response and escalation paths
- [ ] Understand product roadmap and upcoming releases
- [ ] Review customer feedback collection process

---

## Developer Onboarding

- [ ] Development environment setup complete
- [ ] Access to code repositories and project boards
- [ ] Review coding standards and contribution guidelines
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md)
- [ ] Introduction to team and key cross-functional partners
- [ ] Review CI/CD pipelines and testing requirements
- [ ] Complete first small bug fix or contribution
- [ ] Shadow code review process
- [ ] Review architecture documentation
- [ ] Understand branching strategy and PR workflow

---

## Product Manager Onboarding

- [ ] Review product vision and strategy documents
- [ ] Access to product roadmap and backlog tools
- [ ] Review [Project Initiation Guide](octoacme-project-initiation.md)
- [ ] Review [Project Planning](octoacme-project-planning.md)
- [ ] Introduction to stakeholders and executive sponsors
- [ ] Review user research and customer feedback sources
- [ ] Access to analytics and success metrics dashboards
- [ ] Shadow product planning and prioritization sessions
- [ ] Review recent product decisions and trade-offs
- [ ] Understand go-to-market process

---

## Project Manager Onboarding

- [ ] Access to project management tools (Jira, GitHub Projects, etc.)
- [ ] Review all project management process docs in `docs/` folder
- [ ] Introduction to all cross-functional team leads
- [ ] Review active projects, timelines, and risk registers
- [ ] Shadow project meetings (kickoff, planning, retrospectives)
- [ ] Review escalation paths and stakeholder communication templates
- [ ] Access to status reporting tools and templates
- [ ] Understand resource allocation and capacity planning process
- [ ] Review [Risk Management & Communication](octoacme-risks-and-communication.md)
- [ ] Shadow at least one complete project lifecycle

---

## Using This Checklist

1. **For New Hires**: Work with your manager and onboarding buddy to complete the relevant checklist in your first 2-4 weeks.
2. **For Managers**: Customize the checklist based on specific project needs and ensure all items are accessible.
3. **For Updates**: Review and update these checklists quarterly or when significant process changes occur.
4. **For Feedback**: Share improvements or missing items with the Project Management team via the GitHub Issue template.
